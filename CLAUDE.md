## Project Context

### Background
This project is a recreation of the "Magic Bracket" — a single-elimination tournament originally run by Reddit user SaviaWanderer ~8-9 years ago that paired up every Magic: The Gathering card to find a winner. The original bracket ran around the time of Commander 2016 (released 2016-11-11), when there were almost exactly 2^14 = 16,384 unique cards in print.

### Current Goal
Run a new bracket using cards printed **after** Commander 2016 (the "next" 2^14 cards). As of bulk data dated 2026-03-23:
- **~15,800** unique post-C16 cards after all filters (492+ short of 2^14 = 16,384 — roughly 1-2 years of releases away)
- Total unique card count: **32,425** (~200 above Scryfall's search count of 32,226; small unexplained residual)
- **~10%** of post-C16 cards are Universes Beyond (non-Magic IP); identifiable via `security_stamp == 'triangle'`

### Format: Elo-based web applet (not a traditional bracket)
Instead of a fixed single-elimination bracket, the plan is an **Elo rating system** hosted as a web applet:
- Two cards are shown side-by-side; the user picks their favourite
- Elo ratings are updated after each vote
- Matchups are **not random** — cards are paired with similar cards to make early comparisons meaningful
- Elo rating (once built) will also be an input into matchup generation (pair cards at similar Elo levels)

### Matchup generation
Candidates for Card B are generated by `get_candidates(card_name, models, n_neighbors=None)` in `similarity.py`:
- Returns a dict with three keys: `text_heavy`, `balanced`, `struct_heavy`
- A config list is chosen uniformly at random; Card B is then selected from that list
- Same-set candidates are penalised (`SAME_SET_PENALTY = 1.15` distance multiplier) with a fetch buffer of 5 extra neighbors to absorb the reranking

**Four selection modes** (toggled via UI slider in `app.py`):
- **Mode 1 — Similar:** 5 candidates, weighted pick `[0.45, 0.30, 0.15, 0.07, 0.03]`
- **Mode 2 — Diverse:** 15 candidates, uniform pick
- **Mode 3 — Broad:** uniform pick from closest 25% of all cards (balanced model only); pool size `_mode3_n_neighbors = int(len(_card_names) * 0.25)`
- **Mode 4 — Wild:** card_b drawn uniformly from all cards

**Per-session repeat avoidance:** `_session_seen` (server-side dict, session_id → set of seen card names). When card_a has already been seen, 90% chance of re-roll. Both cards in each pair are added to the seen set. Bypassed in closed-loop mode.

## File Structure

### `parse_data.py`
Data loading and filtering pipeline. Key functions:
- `load_card_file(filename)` — loads Scryfall bulk JSON into a DataFrame
- `process_cards(df)` — filters and deduplicates; returns one row per unique card name
- `cards_per_year(df)` — bar chart of first printings per year
- `FILENAME` — hardcoded to current bulk data file

**Filtering applied in `process_cards`:**
- Removes `digital == True` (also removes all Alchemy "A-" cards)
- Removes `reprint == True`
- Removes extra layouts: `token`, `double_faced_token`, `emblem`, `art_series`, `reversible_card`, `sticker`, `planar` (covers Plane and Phenomenon cards from Planechase)
- Removes `set_type` in `{'memorabilia', 'token'}` (catches Dungeons, Role tokens, and novelty helper cards whose layout isn't `token`)
- Removes non-standard card subtypes: cards with `Contraption`, `Attraction`, `Sticker`, or `Scheme` in `type_line`
- Deduplicates by name: sort by `released_at`, then `_is_promo` (non-promo preferred), then `_is_alt_frame` (normal frame preferred over showcase/borderless/extendedart/etched/shatteredglass/inverted), then `id`; keep first

**Output columns:** `id`, `name`, `released_at`, `mana_cost`, `cmc`, `type_line`, `oracle_text`, `colors`, `color_identity`, `keywords`, `produced_mana`, `reprint`, `set_name`, `set`, `set_type`, `rarity`, `artist`, `frame`, `power`, `toughness`, `flavor_text`, `loyalty`, `printed_name`, `layout`, `reserved`, plus one `legal_<format>` column per legality format from Scryfall.

### `similarity.py`
Card similarity model and candidate generation. Key functions:
- `build_candidate_models(df)` — pre-fits NearestNeighbors models; call once at startup, pass result around
- `get_candidates(card_name, models, n_neighbors=None)` — main interface; returns `{'text_heavy': [...], 'balanced': [...], 'struct_heavy': [...]}`. Optional `n_neighbors` overrides the model default (used by modes 2 and 3).
- `find_matches(df, alpha, n_neighbors)` — exploratory helper; returns full ranked match DataFrame
- `show_candidates(card_name, models)` — pretty-prints candidates with alpha labels
- `compare_alphas(df, card_name, ...)` — shows how matches shift across alpha values (tuning helper)

**Similarity approach:** `combined = sqrt(alpha)*struct_L2 + sqrt(1-alpha)*text_L2`
Cosine similarity on combined = `alpha * struct_cosine + (1-alpha) * text_cosine` exactly.

**Structured features (16 base + keywords):**
- CMC (normalised, capped at 16)
- Color identity (binary WUBRG)
- Card types (binary: Creature, Instant, Sorcery, Enchantment, Artifact, Planeswalker, Land, Battle)
- Power / toughness (normalised, capped at 20; 0 for non-creatures)
- Top-60 keyword abilities by frequency (from Scryfall's `keywords` column)
- 41 keyword actions (binary presence in oracle text; see `KEYWORD_ACTIONS` list)

**Text features:** TF-IDF on `oracle_text` (8000 features, unigrams+bigrams, sublinear TF)

**Same-set downranking:** `get_candidates` fetches `n + SAME_SET_FETCH_BUFFER` (5 extra) neighbors, multiplies distance by `SAME_SET_PENALTY = 1.15` for cards sharing a set with the query card, re-sorts, then takes top-n. The models dict stores `_name_to_set` to support this.

**Alpha configs (`ALPHA_CONFIGS`):**
- `text_heavy`: alpha=0.15
- `balanced`: alpha=0.50
- `struct_heavy`: alpha=0.85

**Type overrides (`TYPE_ALPHA_OVERRIDES`):**
- `Planeswalker`: text_heavy=0.40, balanced=0.70, struct_heavy=0.90
  *(Planeswalker oracle text is too sparse for low-alpha matching to be useful)*

### `app.py`
Flask web applet — the main entry point. Run with `python app.py`, then open `http://127.0.0.1:5000`.

- Loads card data and builds similarity models at startup (takes ~30–60s), then initialises Elo ratings
- Serves matchups via `/api/matchup?mode=&closed_loop=` and records votes via `POST /api/vote`
- `/api/matchup/unusual?mode=&closed_loop=` forces card_a to be a Battle or double-faced card (test helper)
- Vote log written to `votes.db` (SQLite); tables: `votes` and `elo_ratings`
- Session IDs assigned via Flask cookie (ephemeral — resets on server restart)
- Card images fetched from Scryfall by dataset ID (`/cards/{id}`) to guarantee original printing; cached in memory as `_card_info_cache`
- `DOUBLE_FACED_LAYOUTS = {'transform', 'modal_dfc'}` — cards in these layouts get both face URLs returned
- `SIDEWAYS_TYPES = {'Battle', 'Room'}` — cards with these type_line substrings are flagged `is_sideways`

**Startup lookup dicts** (all keyed by card name):
- `_name_to_id`, `_name_to_layout`, `_name_to_type`, `_name_to_keywords`, `_name_to_set`, `_name_to_year`

**Key constants:**
- `CANDIDATE_WEIGHTS = [0.45, 0.30, 0.15, 0.07, 0.03]` (mode 1)
- `MODE2_N_NEIGHBORS = 15` (mode 2)
- `MODE3_FRACTION = 0.25` (mode 3 pool = 25% of card list)
- `INITIAL_ELO = 1500.0`, `ELO_K = 32`, `ELO_K_DECAY = 30`
  - Effective K per card: `K * K_DECAY / (K_DECAY + total_games)` — K halves after 30 games
- `SEEN_REROLL_PROB = 0.90`
- `CLOSED_LOOP_SIZE = 25` (**DEV ONLY**)

**`/api/vote` (POST):** records the vote, updates Elo, returns `{elo_a, delta_a, elo_b, delta_b}` only. The next matchup is fetched separately via `/api/matchup` (parallel request from the client).

**`/api/reset_elo` (POST):** resets all ratings to `INITIAL_ELO`. **DEV ONLY — remove before deploying.**

**DEV ONLY features** (marked with comments; remove before production):
- `reset_elo()` / `POST /api/reset_elo`
- `_session_closed_loop` / `_get_closed_loop_pool()` / `CLOSED_LOOP_SIZE`
- Closed-loop mode bypasses similarity, seen-reroll, and mode logic; picks both cards randomly from a fixed 25-card per-session pool

### `templates/index.html`
Single-page web UI served by Flask.

- Displays two card images side by side; clicking one records the vote and loads the next pair
- **Sideways cards** (Battle, Room): displayed rotated 90° clockwise via CSS; frame resized to landscape dimensions
- **Double-faced cards**: hovering shows a centred popup with both face images (280px each)
- **Mode slider** (top-right, fixed, 4 stops): Similar → Diverse → Broad → Wild; active mode name shown in gold with a description line
- **Closed-loop toggle** (top-left, fixed, **DEV ONLY**): pill toggle; when on, restricts matchups to a fixed 25-card session pool
- **Card info below each image**: card name, set + year (italic), current Elo
- **Post-vote Elo update**: after clicking, the Elo line updates in place with the new value and a coloured delta `(+N)`/`(−N)` for 1.5s, then the next pair loads
- **Vote/matchup parallelism**: on click, `POST /api/vote` (Elo) and `GET /api/matchup` (next pair) fire simultaneously; next pair shows after 1.5s (or later if fetch is slow); `isVoting` flag prevents double-clicks
- **Session history** (below cards): pair counter + scrollable log of past votes, prepended, format: **Winner** `Elo ↑` beats **Loser** `Elo ↓`
- **Button row**: "Generate unusual cards" + "Reset Elo" (red, **DEV ONLY**)
